# Contributing to Awesome Fhenix

Thank you for contributing! This document provides guidelines for adding resources.

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-resource`)
3. Add your resource following the formatting guidelines
4. Commit your changes (`git commit -m 'Add amazing resource'`)
5. Push to the branch (`git push origin feature/amazing-resource`)
6. Open a Pull Request

## Submission Guidelines

### Format
- Use format: `- [Name](URL) - Description`
- Descriptions should be concise (one sentence)
- Alphabetize entries within each section

### Quality Standards
- ✅ Must be Fhenix or FHE-related
- ✅ Must be publicly accessible
- ✅ Must provide value to developers
- ✅ No duplicate entries
- ❌ No self-promotion without significant value
- ❌ No paid products without open-source components

### Categories
- **Official Resources**: Links to Fhenix official documentation
- **Libraries**: Open-source code libraries and SDKs
- **Tutorials**: Step-by-step guides and educational content
- **Projects**: Working applications built on Fhenix
- **Research**: Academic papers and technical publications

## Review Process

All submissions are reviewed by maintainers. We may request changes or decline submissions that don't meet our standards.
